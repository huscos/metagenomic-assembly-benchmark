# Comparative Analysis of Metagenomic Assembly and Binning Strategies

This repository hosts a high-performance computing (HPC) benchmarking pipeline designed to evaluate three strategies for recovering Metagenome-Assembled Genomes (MAGs) from complex marine microbiomes. 

The pipeline processes raw paired-end shotgun sequencing reads to compare single-sample processing, multi-sample cross-mapping, and target-driven co-assembly architectures.

### Sample Collection and Dataset Origin
A total of 171 marine water samples were collected along six coastal shelf transects between Höfn and Vík, Iceland, as part of a sampling effort led by Armando Pacheco-Valenciana. Shotgun metagenomic sequencing was performed on the NovaSeq X Plus platform, generating paired-end sequencing reads. 

For the current comparative bioinformatic analysis, a subset of the samples (the second half of the samples) was utilized. This was done due to the dataset’s massive size and the limited computational resources available for benchmarking.

## Workflow Strategies Defined

The benchmark rigorously evaluates three distinct bioinformatic tracks across 85 marine water samples from Iceland:

1. **Single/Single (Mode A):** Single-Sample Assembly with Single-Sample Binning. Reads are mapped exclusively back to their native assemblies using Bowtie2.
2. **Single/Multi (Mode B):** Single-Sample Assembly with Multi-Sample Binning. Leverages cross-sample mapping over target spaces using BWA-MEM to track differential coverage.
3. **Co/Multi (Mode C):** Target-Driven Co-Assembly with Multi-Sample Binning. Algorithmic pairing of highly overlapping community read-pools via `BinChicken`, co-assembled with MEGAHIT, and binned using multi-sample profiles.

### Core Architecture & Tools
* **Quality Control & Profiling:** `Trim Galore` length-filtering, `SingleM` marker-gene taxonomic profiling, and `BinChicken` community-overlap similarity blueprinting.
* **De Novo Assembly:** `MEGAHIT` utilizing succinct de Bruijn graphs.
* **Triple-Algorithm Binning Suites:** Simultaneous partition via `MetaBAT2`, `MaxBin2` (with multi-abundance parameters), and `CONCOCT` (10-kb chunk fragmentation execution).
* **Consolidation & Refinement:** Hybrid cross-bin consolidation using `metaWRAP`'s refinement module to enforce strict selection constraints (Completeness ≥ 50%, Contamination ≤ 10%).
* **Fidelity Evaluation:** Global cluster dereplication via `dRep` (95% ANI species boundary), quality verification via `CheckM`, and taxonomic lineage classification via `GTDB-Tk`.
* **Data Visualization:** Statistical aggregation and plotting utilizing `R` and `ggplot2`.

## Pipeline Execution Roadmap

The scripts inside this repository are organized sequentially to reflect the HPC cluster execution layout and downstream statistical analysis:

### Phase 1: Preprocessing & Profiling (`01_preprocessing_profiling`)
Automates multi-lane FASTQ concatenation, runs length-filtering via Trim Galore, and generates single-copy marker gene tables for overlap matrix calculations.

### Phase 2: Blueprinting & Assembly (`02_assembly_blueprinting`)
Executes targeted predictive pairing configurations to build mapping array lists and automates isolated single-sample contig graph building.

### Phase 3: Mode A - Single/Single (`03_modeA_single_single`)
Maps native reads back to single graphs using Bowtie2 and segments bins using local coverage metrics.

### Phase 4: Mode B - Single/Multi (`04_modeB_single_multi`)
Implements a BWA-MEM mapping framework to build a multi-sample differential coverage matrix across the 20 most genetically overlapping samples defined by the blueprint.

### Phase 5: Mode C - Co/Multi (`05_modeC_co_multi`)
Extracts paired-sample files, co-assembles paired metagenomes with strict memory enforcement targets, cross-maps the recovery matrices, and coordinates the consolidated binner inputs.

### Phase 6: Core Evaluation Rig (`06_evaluation`)
Aggregates the entirety of recovered MAG collections across all modes to perform a competitive dereplication showdown at 95% ANI. Establishes explicit lineage, contamination, and fragmentation benchmarks.

### Phase 7: Visualization & Analysis (`07_visualization`)
The `24_benchmark_visualizations.R` script processes the output matrices from the HPC benchmarking pipeline to evaluate and visualize the performance, taxonomic distribution, and genomic fidelity of the recovered MAGs.

## Data Access & Setup for Visualization

To run the R visualizations (Phase 7), you will need the aggregated summary tables generated by the HPC pipeline (hosted externally).

1. Clone this repository to your local machine.
2. Create a folder named `Data` in the root directory of the repository.
3. Download the necessary evaluation datasets from this link: **https://drive.google.com/drive/folders/15JtIPA5vCGYWyF5JToBOMJTHqMDgMOt9?usp=sharing**
4. Ensure the following files are placed directly inside the `Data` folder:
    * `Master_MAG_Summary_Table.tsv`
    * `Cdb.csv`

**R Dependencies:**
The visualization script requires **R** and the following packages: `dplyr`, `ggplot2`, `readr`, `tidyr`, and `here`.

## Key Benchmark Insights
* **Differential Coverage Advantage:** The Single/Single approach heavily underperforms in raw numbers due to the absence of multidimensional coverage tracks, though it preserves structural accuracy (low fragmentation).
* **The Single/Multi Edge:** Yields the highest quantity of unique species discovery profiles, proving powerful for capturing low-abundance, scattered targets.
* **The Co/Multi Victory:** Recovers MAGs showcasing the highest genomic fidelity (highest checkM completeness medians paired with significantly minimized contamination metrics).
